GWAS data modeling exercise (click to see the original size)

The data is organized the following: users have many phenotypes and many samples, each sample from each user can participate in many GWAS studys and have also some SNPs (which are results for each study). I have considered the table of samples as type of sample and thus the study is related with User_has_Sample, this table has also a Date indicating the data of extraction. I consider that the phenotypes are related to a user independently of the taken sample and the SNPs. SNPs have their statistics and form part of a gene. These genes are studied in an SNParray, which is associated to the Study. I consider SNPs related to each user+sample and also with the study, as the results will depend on the type of study and can be seen in groups (for example, disease or wildtype) and also each user has it's own SNP if results need to be seen individually. Finally, both SNParrays and Studys may have papers of reference. I have used the names that won't be repeted in any element as id for that table instead of generating a default id (for example, in the case of genes.
